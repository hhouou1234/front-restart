# フロントエンジニア未経験ロードマップ

> 目的：未経験から **フロントエンドエンジニアとして収入を得る** 状態になる
> 期限：**2026年4月**
> 方針：参考書・動画に頼らず、**UIを作りながら覚える**

---

## 🎯 最終ゴール（4月）

* HTML / CSS / JavaScript を使って
* **動きのあるUIを実装**できる
* ポートフォリオURLを持ち
* 小さくても **Web系の仕事で収入が発生**している

正社員・高収入は目標にしない。
「今この仕事をしている」と言える状態を作る。

## 🛠 使用ツール一覧

### 開発・学習

* **CodePen**：UI課題・実験場
* **VS Code**：ローカル開発
* **Chrome DevTools**：デバッグ

### 管理・公開

* **GitHub**：コード管理・実績
* **GitHub Pages**：無料で公開（まずはこれ）

> ※ 余裕が出たら選択肢
>
> * **Netlify**：GitHub連携で自動デプロイできる公開サービス
>
>   * 良い点：設定が楽 / 実務でよく使われる
>   * 悪い点：覚えること・管理が増える
>   * 結論：**今は不要、後回しでOK**

### 情報参照（学習用ではなく辞書）

* **MDN Web Docs**
* **CSS-Tricks**

### 仕事探し

* **クラウドワークス**
* **ランサーズ**

## ⛔ やらないこと

## 🧠 運用ルール

1. UI単位で進める
2. 1回90分以内で切る
3. できたら公開
4. わからなくても止まらない
5. スキー・生活優先でOK

---

## 🌅 起きてからの固定ルーティン（作業日）

※ 考えないために **毎回これ**

1. 起きる
2. 水を飲む
3. PCを開く
4. GitHubのこのREADMEを開く
5. 今日やるUIを1つ決める
6. CodePenを開く
7. 30分だけ実装する

### 30分後の分岐

* まだできそう → 最大90分まで続行
* 無理そう → そこで終了（OK）

### 作業後

* READMEのチェックリストに ✔ を入れる
* 次やるUIを1行メモ
