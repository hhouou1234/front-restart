# Roadmap: From Zero to Remote Web Work

## このリポジトリについて
このリポジトリは、  
**未経験・社会経験ほぼなしの状態から、在宅でお金を稼げるようになるまでの記録と実践用リポジトリ**。

目的は「フロントエンジニアになること」そのものではなく、  
**在宅で安定して仕事ができるスキルを身につけること**。

完璧さや継続力は前提にしていない。  
「迷わず次に何をやるか」が分かることを最優先にする。

---

## ゴール（明確化）

### 最終ゴール（3ヶ月想定）
- HTML / CSS / 基本的なJavaScriptでWebページを作れる
- GitHubに制作物と作業過程が残っている
- 小さくても「在宅でお金が発生する仕事」を経験する

### 直近ゴール（1ヶ月）
- Webページの構造を理解できる
- 既存サイトを見てHTML構成を考えられる
- 自分用のシンプルなWebページを1つ作る

---

## 全体ロードマップ

### フェーズ1：基礎理解（1ヶ月目）
**目的：Webの仕組みと型を体に入れる**

- Webサイトを見る
- 構造を言語化する
- HTML / CSSで再現する
- GitHubに文章で記録する

---

### フェーズ2：制作（2ヶ月目）
**目的：仕事に近い形で作れるようになる**

- 架空案件のWebサイト制作
- デザイン → コーディングの流れを経験
- Canva / Figmaなどのデザインツールに触れる
- 制作物をGitHubにまとめる

---

### フェーズ3：仕事化（3ヶ月目）
**目的：在宅で収入を得る経験をする**

- ポートフォリオ整理
- 「何ができるか」を文章で説明できるようにする
- 小さな案件に応募する
- 実務に近い経験を積む

---

## 1ヶ月目：毎日の行動指針（最重要）

### 共通ルール
- 作業時間：2〜3時間でOK
- 完璧を目指さない
- 必ずGitHubに記録を残す（日本語）

---

### Week 1（Day1〜7）
**やること**
- Webサイトを1つ選んで見る
- 以下を文章で書く  
  - このサイトは何のためのページか  
  - どんな構成になっているか  
- HTMLの構造を考える（header / section / footerなど）

**目的**
- Webページの「型」を理解する

---

### Week 2（Day8〜14）
**やること**
- Week1で見たサイトをHTMLで再現
- 見た目は気にしない
- CSSで最低限整える（文字・余白・配置）

**目的**
- HTML / CSSを「知ってる」から「使える」にする

---

### Week 3（Day15〜21）
**やること**
- LP（縦に長いページ）を1つ選ぶ
- 構成・文章・配置を真似して作る
- なぜそう作ったかを文章で残す

**目的**
- 制作の流れを理解する

---

### Week 4（Day22〜30）
**やること**
- 自分用のWebページを作る
  - 自己紹介
  - 学習記録へのリンク
- 毎日少しずつ改善する

**目的**
- 「自分のサイト」を持つ

---

## 記録の考え方（重要）

このリポジトリは日記ではない。  
**考えたこと・理解したこと・判断理由を残すためのドキュメント**。

- 何をやったか
- なぜそうしたか
- 何が分からなかったか

これがそのまま
- 学習の振り返り
- ポートフォリオの説明材料
- 面談や案件応募時の武器になる

---

## 使用ツールの考え方

### PC
- コーディング
- GitHub操作

### iPad
- 記録を書く
- サイトを見る
- 構成を考える
- デザイン確認

iPadは「仕事の準備と管理」、  
PCは「実作業」に使う。
